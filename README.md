# Embedded Smart Home — Environmental Monitor & Actuator Stack (STM32, CAN & Web)


---

## 🚀 Key Features

- **Distributed sensing:** I²C **BME280**-class environmental readout, **PIR** motion on EXTI, and **reed / magnetic door** input, packaged into structured CAN frames.
- **Local actuation:** Arm/disarm via **push button** (deferred to a task with a **binary semaphore**), **buzzer** alarm, **ARM LED**, **HVAC-style LED feedback** with an occupancy timeout, and a **CAN traffic LED** for link health.
- **Industrial-style bus:** Standard **CAN** IDs for sensor broadcasts, actuator status, and remote commands (arm/disarm, temperature setpoint).
- **Edge web UI:** **LwIP**-based **HTTP** server on STM32F7 — dashboard, **JSON** status API, and **POST** command handling that drives outbound CAN frames.
- **Concurrency-aware firmware:** Mutex-protected floats and flags; `memcpy` for packed CAN payloads to avoid misaligned loads on Cortex-M.

## Repository layout

| Folder | Role |
| --- | --- |
| `sender_node/` | Sensor acquisition + periodic CAN transmit |
| `actuator_node/` | FreeRTOS: CAN RX, alarm, climate LEDs, status TX, button task |
| `server node/` | FreeRTOS + LwIP: CAN gateway + HTTP dashboard |

Each folder is a **STM32CubeIDE** project (`.ioc`, HAL, CMSIS-RTOS v2 where applicable).

## CAN Protocol (logical IDs)

| Std ID | Direction | DLC (typ.) | Contents |
| --- | --- | ---: | --- |
| `0x103` | Sender → all | 6 | Door (`'D'`/`'C'`), PIR byte, temperature (`float`) |
| `0x104` | Sender → all | 8 | Pressure (`float`), humidity (`float`) |
| `0x105` | Actuator → server | 6 | Armed, alarm active, target temperature (`float`) |
| `0x200` | Server → actuator | 1 | `0x01` arm / `0x02` disarm |
| `0x201` | Server → actuator | 4 | New setpoint temperature (`float`) |

## Web Dashboard (Server Node)

- **`GET /`** — Single-page **HTML / CSS / JavaScript** dashboard (polished dark UI).
- **`GET /api/status`** — **JSON** snapshot: temperature, humidity, pressure, door, motion, armed, alarm, target setpoint.
- **`POST /api/command`** — Form-encoded body for **arm**, **disarm**, or **setpoint**; server translates to CAN **`0x200`** / **`0x201`**.

The HTTP stack uses **LwIP Netconn** with a small receive reassembly helper so split TCP segments (headers + body) still parse reliably.
